# 足球比赛数据文档分析系统 - 网络版部署文档

> 本文档面向对 Vercel 和 Supabase 不熟悉的用户，按步骤操作即可完成部署。

---

## 一、前置准备

### 1.1 注册账号
1. **GitHub** (https://github.com) - 托管代码
2. **Vercel** (https://vercel.com) - 用GitHub登录
3. **Supabase** (https://supabase.com) - 用GitHub登录

### 1.2 安装软件
1. **Node.js 18+** (https://nodejs.org) - 下载LTS版本安装
2. **Git** (https://git-scm.com)

## 二、配置 Supabase

### 2.1 创建项目
1. 登录 https://supabase.com
2. 点击 **New Project**
3. 填写项目名称（如 `football`）
4. 设置数据库密码（记住它）
5. 选择区域（Southeast Asia推荐）
6. 点击 **Create**

### 2.2 创建数据表
1. 进入项目后，点击左侧 **SQL Editor**
2. 点击 **New query**
3. 打开项目中的 `supabase/schema.sql` 文件
4. 复制全部内容粘贴到编辑器
5. 点击 **Run** 执行
6. 确认无错误，显示 "Success"

### 2.3 创建 Storage Bucket
1. 点击左侧 **Storage**
2. 点击 **New bucket**
3. Bucket名称填 `reports`
4. 勾选 **Public bucket**（公开读取）
5. 点击 **Create bucket**

### 2.4 获取API密钥
1. 点击左侧 **Settings** → **API**
2. 记录以下3个值：
   - **Project URL**: `https://xxx.supabase.co`
   - **anon public key**: `eyJxxx...`
   - **service_role key**: `eyJxxx...`（点击Reveal显示）

---

## 三、部署到 Vercel

### 3.1 上传代码到 GitHub
1. 在 GitHub 创建新仓库（如 `football-web`）
2. 将项目所有文件上传到仓库

### 3.2 导入到 Vercel
1. 登录 https://vercel.com
2. 点击 **Add New** → **Project**
3. 选择刚创建的GitHub仓库
4. Framework Preset 选择 **Next.js**
5. **不要点击Deploy**，先配置环境变量

### 3.3 配置环境变量
在 Vercel 项目的 **Settings** → **Environment Variables** 中添加：

| Name | Value | 说明 |
|------|-------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://xxx.supabase.co` | Supabase项目URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `eyJxxx...` | Supabase anon密钥 |
| `SUPABASE_SERVICE_ROLE_KEY` | `eyJxxx...` | Supabase service_role密钥 |
| `CRON_SECRET` | 自定义随机字符串 | 保护cron接口（可用 `openssl rand -hex 32` 生成） |

### 3.4 部署
1. 回到 Vercel 项目的 **Deployments**
2. 点击 **Redeploy**
3. 等待构建完成（约1-2分钟）
4. 构建成功后，点击域名访问

## 四、验证部署

### 4.1 访问首页
打开 Vercel 分配的域名，应看到比赛列表页面。

### 4.2 测试API
访问 `https://你的域名/api/matches`，应返回JSON格式比赛数据。

### 4.3 测试功能
1. 点击 **刷新数据** 按钮，应成功刷新比赛列表
2. 点击某场比赛的 **报告** 按钮，应生成报告

## 五、常见问题

### Q1: 构建报 "Module not found"
**A:** 确认所有 `src/lib/` 下的6个文件都存在。API路由使用相对路径 `../../lib/xxx`。

### Q2: API报 500 "Missing Supabase environment variables"
**A:** 检查Vercel环境变量是否配置完整（4个变量都要配置）。

### Q3: 报告下载失败
**A:** 检查 Supabase Storage：
1. `reports` bucket 已创建
2. bucket 设置为 Public
3. schema.sql 中的 Storage 策略已执行

### Q4: 比赛列表为空
**A:** 点击"刷新数据"按钮手动刷新。网络版无自动刷新，需手动触发。

### Q5: 批量导入超时
**A:** Vercel免费版函数执行时间限制10秒。批量导入大量ID时可能超时。建议：
- 每次导入不超过5个ID
- 或升级Vercel Pro（60秒超时）

### Q6: docx报告无法转PDF
**A:** Vercel环境无LibreOffice，网络版直接输出docx格式。如需PDF，可在本地转换。
